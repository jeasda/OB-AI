# Phase 3 UX Guide - Qwen Image Edit

## Goal
- ยกระดับประสบการณ์ผู้ใช้ให้พร้อมสำหรับการใช้งานจริงในช่วงเบต้า
- UI ต้องสื่อสารสถานะชัดเจน และรองรับการตัดสินใจเชิงพาณิชย์

## UX Principles
- ภาษาไทยเป็นหลัก ใช้คำเรียบง่าย ไม่เทคนิค
- สถานะต้องชัดเจนและมีคำอธิบาย
- ปุ่มสำคัญต้องเด่นและอยู่ในตำแหน่งที่หาเจอง่าย
- ผลลัพธ์แสดงลายน้ำเพื่อชี้ให้เห็นว่าต้องชำระเงินก่อนดาวน์โหลดจริง

## State-driven UX
- ทุกการเปลี่ยนสถานะมาจาก state machine เท่านั้น
- UI ไม่สร้าง logic ใหม่เอง
- สถานะหลัก: idle, image_uploaded, ready_to_generate, generating, completed, error

## Progress Feedback
- ใช้สถานะจากระบบจริง (queued/running/completed)
- ระบุชัดเมื่อรอคิว GPU
- หลีกเลี่ยง progress ปลอม

## Monetization UX
- แสดงผลลัพธ์พร้อมลายน้ำในหน้าพรีวิว
- ปุ่มดาวน์โหลดระบุว่าเป็นแบบชำระเงิน (ไม่มีลายน้ำ)
- อนุญาตให้เจนใหม่ได้เสมอ

## Error Recovery
- แสดงข้อความที่เป็นมิตร
- มีปุ่มลองอีกครั้งที่ผูกกับ state machine

## Checklist
- [ ] UI แสดงภาษาไทยครบทุกจุด
- [ ] สถานะทุกสถานะมีคำอธิบาย
- [ ] ดาวน์โหลดและเจนใหม่ใช้งานได้
- [ ] ข้อผิดพลาดมีการกู้คืน
- [ ] หน้าใช้งานได้บนมือถือ
